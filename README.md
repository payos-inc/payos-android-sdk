# PayOS Android SDK

Public Android SDK modules for PayOS.

This repository currently contains the PayOS Passkeys SDK module only.

## Requirements

- Android 11 (API level 30) or higher
- Kotlin
- A PayOS link token generated by your server
- A PayOS app routing ID issued by PayOS
- Android App Links configured for `https://link.payos.ai`

## Installation

Add this repository as a source dependency, for example as a Git
submodule under `vendor/payos-android-sdk`, then include the passkeys
module in your app's `settings.gradle`:

```groovy
include ':payos-passkeys'
project(':payos-passkeys').projectDir = file('vendor/payos-android-sdk/passkeys')
```

Add the module dependency in your app's `build.gradle`:

```groovy
dependencies {
    implementation project(':payos-passkeys')
}
```

Then configure the SDK:

```kotlin
import ai.payos.PasskeysConfiguration
import ai.payos.PayOS
import ai.payos.PayOSConfiguration
import ai.payos.passkeys.CardBrand

PayOS.configure(
    context = applicationContext,
    configuration = PayOSConfiguration(
        linkToken = "lnk_...",
        sandbox = true,
        passkeys = PasskeysConfiguration(appRoutingId = "androidapp_rt_...")
    )
)

val card = PayOS.passkeys.card(cardId)

if (card.brand == CardBrand.MASTERCARD && card.nativeAuthenticationAvailable == true) {
    val result = PayOS.passkeys.authenticate(
        activity = activity,
        paymentIntentId = paymentIntentId,
        paymentId = paymentId
    )
} else {
    // Use your existing PayOS web bridge / Payment Sheet.
}
```

The passkeys client APIs are suspending Kotlin functions. Call them
from a coroutine scope that stays alive while Chrome Custom Tabs is
open.

## App Links

Your app must claim its PayOS callback path:

```xml
<intent-filter android:autoVerify="true">
    <action android:name="android.intent.action.VIEW" />
    <category android:name="android.intent.category.DEFAULT" />
    <category android:name="android.intent.category.BROWSABLE" />

    <data
        android:scheme="https"
        android:host="link.payos.ai"
        android:path="/passkey-callback/YOUR_APP_ROUTING_ID" />
</intent-filter>
```

PayOS must add your Android package name and SHA-256 signing
certificate fingerprint to its native app registry before
`link.payos.ai` can open your app.

Forward App Link intents to PayOS from your launcher activity:

```kotlin
override fun onCreate(savedInstanceState: Bundle?) {
    super.onCreate(savedInstanceState)
    if (PayOS.handleIntent(intent)) {
        lifecycleScope.launch {
            PayOS.passkeys.resumePendingSession()
        }
    }
}

override fun onNewIntent(intent: Intent) {
    super.onNewIntent(intent)
    setIntent(intent)
    if (PayOS.handleIntent(intent)) {
        lifecycleScope.launch {
            PayOS.passkeys.resumePendingSession()
        }
    }
}
```
